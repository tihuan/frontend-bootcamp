Introduction

The purpose of this demo is to show case a few common HTML elements that you will likely use in a real project. Later in the exercise, you will be able to play with the elements to understand how they work!

For now, just read through each of the following category to get familiar with their special purpose and usage, so you have a great mental model in place to help guide you when to use what element.

Document Metadata

Metadata contains information about the page. This includes information about styles, scripts and data to help software (search engines, browsers, etc.) use and render the page. Metadata for styles and scripts may be defined in the page (inline) or link to another file that has the information.

Inline: When inlining a script or style is needed for a special reason, such as performance, you can do so via: 

          <style>
            hr {
              margin: 40px;
            }
          </style>

          <script>
            function helloWorld() { alert('Hello world!') }
          </script>

The following is an example of an HTML page's <head> section: 

          <html>
            <head>
              <title>Intro to HTML</title>
              <link rel="stylesheet" href="./style.css" />

              <style>
                hr {
                  margin: 40px;
                }
              </style>
            </head>
          </html>

Block Text content

Use HTML text content elements to organize blocks or sections of content placed between the opening <body> and closing </body> tags. Important for accessibility (a11y) and SEO, these elements identify the purpose or structure of that content.

In the following examples, you will see that the browser applies minimal styling for each element type by default, so the HTML page doesn't look too ugly if no custom styles are provided.

For example, <p> is styled with the following by Chrome: 

          p {
            display: block;
            margin-block-start: 1em;
            margin-block-end: 1em;
            margin-inline-start: 0px;
            margin-inline-end: 0px;
          }
And <div> only gets: 
          div {
            display: block;
          }

Additionally, <div> and <p> signal different content purpose to SEO, a11y, and developers. When <p> is used, it signals that the content is a paragraph, whereas <div> is only a generic block level container that doesn't convey any content purpose.

Pre

Respects spaces and new lines:

This tag defines preformatted text and is displayed in a fixed-width font (usually Courier). It preserves both spaces and line breaks as shown below: 

        // This is a pre tag            --           <-- It respects spacing and tabs

        <ul>
          <li>Unordered</li>
          <li>list</li>
          <li>items</li>
        </ul>

In comparison, the same code block wrapped in <div> looks like this:

// This is a div tag -- <-- It does not respect the spacing, tabs, and linebreaks <ul> <li>Unordered</li> <li>list</li> <li>items</li> </ul>
